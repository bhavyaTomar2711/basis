**What I learned:**
- React 19's `set-state-in-effect` rule is too strict for the legitimate "read external storage at mount" pattern. `useSyncExternalStore` works for cross-tab listeners but localStorage doesn't fire `storage` events for same-tab writes, so it can't be used as the source of truth for an editable form. The simple `useState + useEffect to load + useEffect to write` pattern is correct here; rule demoted.
- `Math.random()` in render is genuinely a bug — the audit ID changes on every re-render, so footnote anchors break. Replaced with a deterministic djb2-style hash of the input. Same input → same id, every render.
- Framer Motion would add ~50KB just for one count-up. 30-line vanilla rAF loop does the same job and is easier to reason about under `prefers-reduced-motion`.
- Tailwind v4's `@theme inline` is great for token → utility mapping. The `bg-grid` + `bg-grid-mask` utilities took 8 lines of CSS, render at 0 cost, and look like the Credex pattern.

- Reduced-motion-respecting count-up uses a synchronous setState to "snap" to value on the reduced-motion path. Lint warns. Functionally correct but the rule is right that it's a code smell. Day 4 polish: refactor to use `useRef` + DOM mutation so no setState is needed at all.

**What I learned:**
- The Resend sandbox sender (`onboarding@resend.dev`) only delivers to the email the account was registered with — not the recipient address. Caught this reading the docs before testing. **What I learned:**
- Sandboxed iframes with `sandbox=""` are stricter than I remembered — even *same-origin* documents loaded into one render as unique-origin and can't fetch their own CSS. That's why the share page rendered unstyled when the email's "View full report" link tried to load it inside the preview. `allow-top-navigation-by-user-activation` is the narrowest fix that lets a *user-initiated* click break out, without re-enabling scripts or same-origin reads inside the frame.
- Gemini's `finishReason` is the single most important field on the response object and I wasn't reading it. The SDK exposes it at `response.candidates[0].finishReason`. STOP / MAX_TOKENS / SAFETY / RECITATION are all distinct cases; only STOP is "the model finished its sentence." Reading just `response.text` and trusting it was a v1-shortcut that bit me. Won't happen again.
- The economics math forces a hard read on Credex CTA conversion. If `tier=high` audits convert to Credex conversations at <12%, the entire ARR thesis falls apart and Basis is structurally a free SaaS tool, not a lead source. Writing `ECONOMICS.md` was the exercise that made this obvious — I'd been treating "Credex back-loop" as a free lunch in my head. It isn't free; it's the load-bearing assumption.
